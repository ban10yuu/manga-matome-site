# Lessons Learned

## 2026-03-10: 記事作成後の品質チェック必須
- **問題**: 記事を大量生成した後、チェックせずに放置した
- **対策**: 記事作成後は必ず以下を確認する
  1. ビルドテスト（コンパイルエラー）
  2. slug重複チェック（全記事ユニーク）
  3. 日付分布チェック（公開ペース確認）
  4. コンテンツ品質（HTMLクラス使用、セクション数、トーン統一）
  5. 開発サーバーでの表示確認（レイアウト崩れ）
- **ルール**: 作りっぱなしにしない。生成物は必ず検証してから完了とする。

## 2026-03-10: 外部リンク（ISBN/URL）は必ず実サイトで検証する

**問題**: 20作品中14作品のAmazon ISBN-10が誤っており、全く別のマンガにリンクしていた。ユーザーの信頼を大きく損なう重大バグ。

**原因**: ISBNを推測/生成しただけで、実際のAmazon商品ページを確認しなかった。

**ルール**:
1. **ISBN/ASIN/商品コードは絶対に推測しない** → 必ず実サイト（Amazon等）で検証する
2. アフィリエイトリンクを設定したら、**全リンクをWebFetchで実際にアクセスして商品名を確認**する
3. 収益サイト構築時のチェックリストに「全外部リンクの動作確認」を追加
4. 画像URLがISBNベースの場合、画像が正しい表紙かも確認する
